# v1.1 architecture-convergence

这是“宅宅能量条系统”的 v1.1 架构收口快照。

## 本版重点

- Core 改为命令 Facade：UI 通过 `GameCommand` 调用 `GameCore.execute()`。
- Core 返回 `OperationResult`，展示消息统一通过 `GameEvent` 传递。
- 新增 Service 层，业务结算从 UI/Core 中拆出。
- 新增 Repository 层，玩家存档和日志 IO 统一放到 `repositories/player_repository.py`。
- `PlayerService` 收口为唯一玩家状态修改入口，不再负责 JSON 或日志 IO。
- `Player` 保持为纯状态实体，兼容现有 `save.json` 结构。
- `GameState` 收口为 UI ViewModel，只包含展示所需字段。
- UI 只负责发送命令、处理事件和渲染状态。
- 增加架构守卫测试，防止直接修改玩家状态、PlayerService 回流 IO、UI 穿透 Core 内部结构。

- 项目说明：`README.md`

- `save.json`：本地玩家存档
- `log.txt`：本地行动日志
- `.idea/`：IDE 配置
- `.venv/`：本地虚拟环境

这些文件属于本机运行状态或构建产物，不作为 v1.1 项目历史记录。
